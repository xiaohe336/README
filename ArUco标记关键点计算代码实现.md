# 基于 ArUco 的开关面板自动标注工具实战指南

本项目利用视觉与几何透视原理，自动计算并精准标注开关面板的四个关键角点。以下是快速上手流程：

---

## 第一步：物理环境准备

必须严格按以下规格布置：

* **ArUco 标记**：打印 **5x5 字典**（`DICT_5X5_1000`）、**ID 为 1** 的 ArUco 二维码。
* **面板尺寸**：准备标准 **8cm × 8cm** 正方形开关面板。
* **共面要求**：开关面板与 ArUco 码必须贴在**同一绝对平整的平面（Z=0 平面）**上。
* **几何校准**：以 ArUco 二维码的**左下角**为物理原点，向左平移精确的 **10cm**，严丝合缝地对齐开关面板的**右上角**。

---

## 第二步：环境配置

确保安装 Python 3.8+，执行以下命令安装依赖：

```bash
pip install opencv-python opencv-contrib-python numpy
```

---

## 第三步：运行与操作

连接 1080p 90Hz 摄像头。程序启动后，会自动将 640×480 的基础标定内参按比例平滑缩放至 1920×1080。在项目根目录下运行：

```bash
python aruco_cam.py
```

**快捷键控制**：

* **`s` 键**：单帧保存（生成高清原图与 JSON 标注）。
* **`c` 键**：切换手动触发与自动连续抓拍模式。
* **`p` 键**：暂停或继续画面流，用于检查关键点贴合度。
* **`q` 或 `ESC` 键**：安全退出程序。

---

## 第四步：数据清洗与质量控制

若遇到系统拒绝保存画面，属于以下正常保护机制触发：

* **画面抖动**：系统采用“3帧快速移动平均滤波”消除视觉抖动。请保持摄像头稳定约 0.5 秒。
* **视角遮挡**：ArUco 码被遮挡或反光，无法识别出全部 4 个关键点，该帧被丢弃。
* **边缘保护**：当关键点移动到 1920×1080 画面的最边缘或超出边界时，为防止目标被裁切，该帧被丢弃。

---

## 第五步：数据集输出格式

系统会在指定的输出目录下生成标准的 LabelMe 数据集结构：

```text
dataset_1080p90/
├── images/
│   └── frame_20260401_120000.jpg   # 1080p 原图
└── labels/
    └── frame_20260401_120000.json  # 对应的 4 个关键点坐标标注
```

**JSON 文件规范**：

1.  `imagePath` 自动采用标准化相对路径（如 `images/frame_xxxx.jpg`）。
2.  `imageWidth` 和 `imageHeight` 始终与捕获的 1080p 实际物理像素保持一致。
3.  关键点标签定义明确：`1` (左上)、`2` (右上)、`3` (右下)、`4` (左下)。
