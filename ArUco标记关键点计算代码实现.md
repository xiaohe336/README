# 🎯 基于 ArUco 的开关面板自动标注工具 (新手实战指南)

无需手动逐一框选或打点，本项目利用摄像头视觉与几何透视原理，自动计算并在图像中精准标注开关面板的四个关键角点。本指南将手把手教你完成从物理环境搭建到数据集生成的全过程。

---

## 🛠️ 第一步：物理环境准备（数据准确性的基石）

计算机视觉对真实世界的物理尺寸极其敏感。请务必严格按照以下规格布置你的实验场地：

### 1. 打印与准备 ArUco 标记
* 准备一张 **5x5 字典**（`DICT_5X5_1000`）、**ID 为 1** 的 ArUco 二维码。
* 确保打印出的二维码黑色方块区域边长符合你在代码中设定的物理尺寸。

### 2. 严苛的粘贴规则（核心步骤）
* 准备一个标准的 **8cm × 8cm** 正方形开关面板。
* 将开关面板与 ArUco 码贴在**同一个绝对平整的平面（Z=0 平面）**上，不能有任何凸起或倾斜。
* **空间几何校准**：以 ArUco 二维码的**左下角**为物理原点，向左平移精确的 **10cm**，必须严丝合缝地对齐开关面板的**右上角**。

---

## 💻 第二步：软件与环境配置

确保你的电脑已安装 Python 3.8 或以上版本。打开命令行终端，执行以下命令安装运行所需的依赖库：

```bash
pip install opencv-python opencv-contrib-python numpy
```
> **注意**：必须安装 `opencv-contrib-python`，因为它是解析 ArUco 标记的核心模块。

---

## 🚀 第三步：运行程序与快捷键操作

将你的 1080p 90Hz 高帧率摄像头连接至电脑。程序启动后，系统会自动将 640×480 的基础标定内参按比例平滑缩放至 1920×1080，确保高清分辨率下的投影精度。

在项目根目录下运行启动命令：

```bash
python aruco_cam.py
```

### 🎮 交互控制台

当摄像头画面显示后，你可以通过键盘进行实时控制：

* **`s` 键（单帧保存）**：当关键点稳定且完全贴合时按下，瞬间捕捉当前高清帧并生成对应的 JSON 标注文件。
* **`c` 键（模式切换）**：在“手动触发”与“系统自动连续抓拍”模式之间一键切换。
* **`p` 键（画面冻结）**：暂停或继续画面流，方便仔细检查关键点投射位置是否精准。
* **`q` 键 / `ESC` 键**：安全停止并退出程序。

---

## 🛡️ 第四步：防坑指南（为什么画面没保存？）

在操作时，你可能会遇到系统“拒绝保存”的情况。这是内置的**数据清洗与质量保护机制**在发挥作用：

* **画面出现轻微抖动**：请保持摄像头稳定约 0.5 秒。系统底层采用了一套轻量级的“3帧快速移动平均滤波”算法。它比卡尔曼滤波响应更快，能在消除视觉抖动的同时，极速锁定面板四角。
* **被判定为“无效帧”丢弃**：
  * **视角遮挡**：只要 ArUco 码的任意部分被遮挡或反光，无法识别出全部 4 个关键点，系统将拒绝保存错误数据。
  * **触发边缘保护**：为防止模型训练时目标被意外裁切，只要关键点移动到了 1920×1080 画面的最边缘（或超出边界），该帧画面会被系统静默丢弃。

---

## 📁 第五步：查收标准数据集

顺利运行后，系统会在你指定的输出目录（例如 `dataset_1080p90`）下，自动分类生成标准的 LabelMe 数据集结构，可直接喂给 YOLO 等深度学习模型：

```text
dataset_1080p90/
├── images/
│   ├── frame_20260401_120000.jpg   <-- 仅保存完整的 1080p 原图
│   └── ...
└── labels/
    ├── frame_20260401_120000.json  <-- 包含 4 个关键点坐标的标注文件
    └── ...
```

### JSON 标注文件说明
输出的 JSON 文件严格遵循 LabelMe 5.5.0 标准：
1. `imagePath` 自动采用标准化相对路径（如 `images/frame_xxxx.jpg`），无论如何迁移文件夹都不会导致路径报错。
2. 文件内记录的 `imageWidth` 和 `imageHeight` 始终与捕获的高清图片实际物理像素保持一致。
3. 关键点标签定义明确：`1` (左上)、`2` (右上)、`3` (右下)、`4` (左下)。
