## Jetson 平台 TensorRT 部署与推理

* **Jetson 工作路径**：`/home/admin1/yolo`
* **前提条件**：请将训练好的 `best.pt`、转换脚本 `pt_To_engine.py` 以及推理脚本 `pnp_video6.py` 全部放置在该工作路径下。

### 1. 启动 YOLO 运行环境 (Docker)
在 Jetson 的主终端中，运行以下命令映射摄像头、激活 GPU 并进入容器：

```bash
sudo docker run -it \
  --device=/dev/video0 \
  --ipc=host \
  --rm \
  --runtime=nvidia \
  -v /home/admin1/yolo:/workspace/yolo \
  -w /workspace/yolo \
  ultralytics/ultralytics:latest-jetson-jetpack6
```
*(提示：命令执行后，终端提示符将变为 `root@xxxx:/workspace/yolo#`，代表你已成功进入容器内部环境。接下来的命令都在此终端内执行。)*

### 2. 导出 TensorRT 加速引擎
利用 Jetson 的 GPU，将标准的 `.pt` 模型转换为极速推理的 `.engine` 格式（已开启 FP16 半精度加速，专为 Jetson 优化）。

```bash
python3 pt_To_engine.py
```
* **预期结果**：在对应权重目录下生成 `best.engine` 文件。
* **注意**：该步骤**只需执行一次**，之后只要你不更换模型权重，就可以直接跳过这一步。

### 3. 实时 3D 姿态推理与测距
调用外接摄像头，进行 TensorRT 极速推理、PnP 空间测距解算、绘制 3D 坐标轴与九宫格透视区域。

```bash
python3 pnp_video6.py
```

**交互配置指引**：
程序启动后，按终端提示输入（或直接回车使用默认值）：
1. **摄像头ID**：默认 `0`（如有多个摄像头可改 `1` 或 `2`）。
2. **输出视频路径**：默认 `output_video_9grid.mp4`（程序会以恒定 30 FPS 录制，防止播放时倍速失真）。
3. **录制时长**：输入想要的秒数，或留空以便随时 `Ctrl+C` 手动停止。
4. **平滑窗口**：默认 `5` 帧。用于时序滤波，消除关键点微小抖动，让 3D 坐标系更加稳定。

### 4. 退出环境与查看结果
测试与录制完毕后，在终端输入以下命令退出容器：

```bash
exit
```
安全退出 Docker 容器后，你将回到 Jetson 的本地系统。录制好的视频已自动保存在 `/home/admin1/yolo` 目录下，可直接拷贝出来查看测距精度与九宫格的透视映射效果。
